# 3D Style Transfer Mode

Use for ordinary FigMirror output where user data should inherit the reference's
3D visual grammar.

## Authority Split

L1 owns representation family, camera class, panes/grid posture, mark or surface
treatment, colorbar/legend placement, typography scale, and export readability.
User data owns values, ranges, counts, categories, coordinates, labels, and
valid encodings. Never copy L1 hidden formulas, coordinates, category identities,
or series counts unless the user supplied them.

## Flow

Start with a conservative render in the same 3D family. Move only the cues needed
for the user's data to read cleanly: depth order, density, alpha, mesh/facets,
relief, color semantics, or panel spacing. If a change shrinks the subject,
changes chart family, collides labels, or drops user data, keep the conservative
render and repair locally.

Do not load strict scorecards, candidate pools, or control-replacement language.

## Review

The render is acceptable when it remains visibly 3D, keeps the reference's
family/camera/export grammar, and maps that grammar onto user-owned data. Mark
`off` if it copies reference data, becomes a different chart class, loses user
data, or fails at final export size.
